> Looking for a setup that doesn't require Figma Desktop? See [Cloud mode](/docs/get-started/cloud).

## Config path

- macOS: `~/Library/Application Support/Claude/claude_desktop_config.json`
- Linux: `~/.config/Claude/claude_desktop_config.json`
- Windows: `%APPDATA%\Claude\claude_desktop_config.json`

This is the file Claude Desktop reads at launch to discover MCP servers. The setup CLI writes the `mcpServers.figma` entry into it.

## What `setup` writes

```json
{
  "mcpServers": {
    "figma": {
      "command": "npx",
      "args": ["-y", "@bromso/figma-mcp"]
    }
  }
}
```

`setup` preserves any sibling entries in `mcpServers` and any other top-level keys in the file. The write is atomic (temp file plus rename), so a partial write can never corrupt the config.

## Reload behavior

Quit and relaunch the Claude Desktop app — the config is only read at process start.

## Verify

```bash
figma-mcp doctor --json | jq '.checks[] | select(.id == "ai-client-configs")'
```

This confirms the entry is present and that the binary on `PATH` matches the configured command.

## Troubleshooting

- Entry written but not visible — see [config drift](/docs/reference/troubleshooting#ai-client-configs-config-drift).
- Tool call hangs — see [plugin not paired](/docs/reference/troubleshooting#plugin-pairing-plugin-not-paired).
- Daemon errors after a Claude Desktop update — see [stale lockfile](/docs/reference/troubleshooting#daemon-liveness-stale-lockfile-dead-daemon).
