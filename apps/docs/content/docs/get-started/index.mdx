## What you need

- Bun OR Node ≥ 20.10 (the binary runs under either).
- Figma desktop (the bridge plugin runs in Figma's plugin sandbox; not in the browser).
- An MCP-aware AI client: Claude Code, Claude Desktop, Cursor, Windsurf, or VS Code Copilot. See [per-client guides](/docs/clients) for the matrix.

## Step 1: Install figma-mcp

```bash
npx @bromso/figma-mcp setup
```

`setup` auto-detects the AI clients on your machine, prints a table of the configs it will write, and writes them atomically. Use `--dry-run` to preview without touching anything, `--client <id>` to scope to a single client (useful when you only want to wire one of several detected clients), and `--cloud` to use the hosted relay instead of local IPC.

> If your AI client is on a different machine than Figma Desktop, see [Cloud mode](/docs/get-started/cloud) instead.

## Step 2: Open Figma and import the bridge plugin

```bash
figma-mcp setup --open-figma
```

This opens the OS file picker at the bundled `manifest.json`. In Figma Desktop, go to **Plugins → Development → Import plugin from manifest** and pick the file. The plugin appears in your dev plugins list, ready to run.

## Step 3: Run the bridge plugin in Figma

Open any Figma file and run **Plugins → Development → Bridge** from the menu. The plugin window opens and pairs with the local daemon over loopback WebSocket; the handshake is silent on success and the plugin shows a **Connected** indicator.

## Step 4: Verify

```bash
figma-mcp doctor
```

`doctor` reports daemon liveness, plugin pairing, AI-client config drift, recent daemon errors, IPC socket conflicts, and FIGMA_API_KEY presence.

A clean run prints all six checks as `ok`. The sixth check (`figma-api-key`) is a `warn` if you haven't set `FIGMA_API_KEY` — that's fine for local-mode. See [troubleshooting](/docs/reference/troubleshooting) for any non-`ok` checks.

## Step 5: Issue your first tool call

Open your AI client and prompt: *"Using figma-mcp, call `bridge_status` and tell me whether the bridge plugin is paired."* The client invokes the `bridge_status` tool over MCP; the daemon reports daemon liveness and plugin pairing as a small JSON object.

If your client surfaces tool transcripts (Claude Code does, in the inline tool-use block), inspect the request and response there. From here, ask for selection details, mutate fills, or any other tool the catalog exposes — your AI client discovers them via `tools/list`.
